Control de Estructuras

Comandos de Preparación (DDL)

Gestionando el Servidor

Para poder manipular datos, el administrador debe saber cómo crear y eliminar los contenedores (Bases de Datos y Tablas).

CREATE
Creación

Establece nuevas bases de datos o tablas desde cero.

DROP
Eliminación

Borra permanentemente estructuras completas.

TRUNCATE
Limpieza

Vacía el contenido de una tabla sin borrar su estructura.

RENAME
Renombrar

Cambia el nombre de una tabla existente.

Administración de Estructura 
-- 1. Crear nueva base de datos
CREATE DATABASE sistema_ventas;

-- 2. Eliminar base de datos (¡CUIDADO!)
DROP DATABASE sistema_ventas;

-- 3. Eliminar una tabla específica
DROP TABLE Clientes;

-- 4. Vaciar una tabla sin borrarla
TRUNCATE TABLE Ventas_Temporales;

Gestión de Contenido

Lenguaje de Manipulación (DML)

Comandos de Acción Directa

La manipulación de datos requiere precisión. Un comando mal ejecutado sin un filtro WHERE puede alterar o borrar toda la base de datos de la empresa.

1. Carga de Datos (INSERT)

Se utiliza para poblar las tablas. Podemos insertar un solo registro o múltiples filas en una sola instrucción.

2. Mantenimiento (UPDATE)

Para corregir errores o actualizar estados (ej. cambiar el precio de un producto).

3. Depuración (DELETE)

Elimina filas específicas. Es una acción irreversible en la mayoría de los casos.

Laboratorio SQL: Manipulación Real 
-- Inserción múltiple de productos
INSERT INTO Productos (id, nombre, precio, stock) 
VALUES 
(1, 'Teclado Gamer', 450.00, 15),
(2, 'Mouse Óptico', 180.50, 30);

-- Actualización con cálculo
UPDATE Productos 
SET precio = precio * 1.15 
WHERE stock < 10;

Consultas y Reportes (DQL)

Extracción y Agregación

No basta con ver los datos, hay que resumirlos. SQL ofrece **Funciones de Agregado** para obtener estadísticas instantáneas.

COUNT()

Cuenta registros.

SUM()

Suma valores.

Flujo de Consulta
1. Identificar Tabla (FROM)
2. Filtrar Filas (WHERE)
3. Elegir Columnas (SELECT)
-- Reporte de ventas totales por día
SELECT fecha, SUM(total) AS ingreso_diario 
FROM Ventas 
GROUP BY fecha;

Semana de Automatización

Disparadores (Triggers)

Lógica de Eventos

Los disparadores permiten que la base de datos se "defienda" o "reaccione" sola. Usamos las palabras clave OLD y NEW. 

-- Auditoría de Precios
CREATE TRIGGER audit_cambio_precio
BEFORE UPDATE ON Productos
FOR EACH ROW
BEGIN
    INSERT INTO Historial (id_prod, anterior)
    VALUES (OLD.id, OLD.precio);
END;

Abstracción y Seguridad

Las **Vistas** son capas de seguridad. Permiten que un empleado vea el inventario pero no el costo de compra del proveedor. 

-- Crear vista de catálogo
CREATE VIEW v_CatalogoPublico AS
SELECT nombre, marca, precio_venta 
FROM Productos;
