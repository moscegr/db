Análisis y Requerimientos

Duración: 7 Horas

Guía Teórica: El Arte de Escuchar

Para construir una base de datos, primero debemos identificar los **Requerimientos Funcionales**. Esto implica entrevistar al usuario final para saber qué "acciones" realizará el sistema.

1

Identificación de Actores: ¿Quiénes alimentan los datos? (Vendedores, Alumnos, Administradores).

2

Flujo de Información: ¿Qué datos entran y cuáles deben salir como reportes?

3

Restricciones: ¿Hay datos que no pueden repetirse? ¿Hay valores obligatorios?

Términos Clave S1

Entidad (Entity)
"Representa un objeto real o abstracto del cual se recopilan datos. En SQL se convertirá en una TABLA."
Atributo (Attribute)
"Cada característica de la entidad. En SQL se convertirá en una COLUMNA."

La Regla de Oro:
Cardinalidad

Define cuántas veces una instancia de una entidad se relaciona con otra. Es el paso más crítico del diseño.

1:1

Uno a Uno

Ejemplo: Un País tiene una sola Capital.

1:N

Uno a Muchos

Ejemplo: Un Cliente puede tener muchos Pedidos.

N:M

Muchos a Muchos

Ejemplo: Muchos Libros son escritos por muchos Autores.

Duración: 7 Horas

¿Por qué Normalizar?
Eliminando Anomalías

La normalización es un proceso sistemático para evitar la duplicidad de datos y garantizar que cada dato se guarde **exactamente en un solo lugar**.

Fase 1
1FN: Atomicidad

"Prohibido tener múltiples valores en una celda o columnas repetitivas."

INCORRECTO: [Teléfonos: 747123, 747456]
Fase 2
2FN: Dependencia Funcional

"Todos los datos deben depender al 100% de la Llave Primaria."

Fase 3
3FN: No Transitividad

"No puede haber columnas que dependan de otras columnas que no sean llaves."

Diseño Físico y Lenguaje SQL

Duración: 7 Horas
MySQL / MariaDB 
-- 1. CREACIÓN DE LA ESTRUCTURA
CREATE TABLE Alumnos (
    id_alumno  INT PRIMARY KEY,
    curp       VARCHAR(18) UNIQUE NOT NULL,
    nombre     VARCHAR(100) NOT NULL,
    id_carrera INT,
    FOREIGN KEY (id_carrera) REFERENCES Carreras(id)
);

-- 2. MODIFICACIÓN (EJ: AÑADIR EMAIL)
ALTER TABLE Alumnos ADD correo VARCHAR(50);

Sintaxis Explicada (DDL)

INT

Números enteros (Matrículas, cantidades, IDs).

VARCHAR(N)

Cadenas de texto variables con un límite máximo (N).

NOT NULL

Restricción que obliga a que el campo nunca quede vacío.

Del Diagrama
al Código Final

En esta semana aprenderás a usar el Lenguaje de Definición de Datos (**DDL**). Es el momento donde el diseño conceptual se vuelve una realidad funcional en el servidor.

Comando 1.3.1
CREATE TABLE

Crea la estructura física de la tabla, definiendo nombres de campos, tipos de datos y llaves.

Comando 1.3.2
ALTER & DROP

Permiten evolucionar la base de datos (añadir columnas, borrar tablas viejas).
